# LLM 监控平台方案

1. 后端日志采集

- 在`backend/src/runtime/llm-player-executor.ts`与相关执行流程中注入记录逻辑，将system prompt、应用prompt、游戏名、回复状态等写入PostgreSQL新表（新增迁移与Repository/Service）。

2. API 层实现

- 新增如`GET /api/v1/llm-logs`、`GET /api/v1/llm-logs/:id`等路由，支持状态过滤与分页，返回最新日志及未回复/重试失败信息。

3. 独立前端应用

- 在`llm-monitor/`建立Vite+React页面（无鉴权），调用后端API展示日志列表详情，强调等待/失败状态的可视化。

4. 基础设施集成

- 在`docker-compose.yml`新增前端服务配置（构建`llm-monitor`），配置环境变量并确保通过Nginx访问；与现有Nginx转发整合。
