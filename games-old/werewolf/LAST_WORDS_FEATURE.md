# 狼人杀遗言功能说明

## 功能概述

成功为狼人杀游戏添加了**遗言环节**功能。当任何玩家因任何原因死亡时，都会触发遗言阶段，让即将死亡的玩家发表最后的话。

## 触发场景

遗言环节会在以下所有死亡情况下触发：

1. **夜晚死亡**
   - 被狼人杀害
   - 被女巫毒杀

2. **白天死亡**
   - 被投票放逐

3. **猎人开枪**
   - 被猎人带走

## 实现细节

### 1. 类型定义 (types.ts)

- 添加新的阶段类型：`'last_words'`
- 添加遗言记录接口：`LastWordsRecord`
- 在游戏状态中添加：
  - `lastWordsHistory`: 存储所有遗言记录
  - `lastWordsContext`: 管理遗言阶段的上下文信息

### 2. 死亡处理逻辑 (phases.ts)

修改了 `applyDeaths` 函数：
- 在玩家真正死亡之前，先进入遗言阶段
- 设置 `pendingRoles` 为所有即将死亡的玩家
- 保存死亡记录和恢复阶段信息到 `lastWordsContext`

### 3. 游戏流程 (index.ts)

添加了以下功能：

#### 遗言动作处理
- `applyLastWordsAction`: 处理玩家发表的遗言
- 验证遗言内容不为空
- 记录遗言到 `lastWordsHistory`

#### 遗言后恢复
- `resumeAfterLastWords`: 遗言阶段结束后的流程恢复
- `applyDeathsDirectly`: 实际应用死亡效果
- 正确处理猎人技能触发和游戏流程继续

### 4. 视角生成 (perspective.ts)

- 添加遗言阶段的提示消息
- 显示待发表遗言的玩家列表
- 显示当天的遗言历史记录
- 为观战者提供遗言阶段的视角

## 游戏流程示例

### 单人死亡流程

```
1. 夜晚阶段结束，狼人杀害了玩家A
2. 进入遗言阶段 (last_words)
3. 玩家A发表遗言
4. 遗言结束，玩家A正式死亡
5. 检查是否触发猎人技能
6. 进入白天讨论阶段
```

### 多人同时死亡流程

```
1. 夜晚阶段，狼人杀害玩家A，女巫毒杀玩家B
2. 进入遗言阶段 (last_words)
3. 玩家A发表遗言
4. 玩家B发表遗言
5. 所有遗言发表完毕
6. 玩家A和B正式死亡
7. 按顺序检查是否触发猎人技能
8. 进入白天讨论阶段
```

## 使用方法

### 发表遗言

当玩家进入遗言阶段时，会看到提示：
- **当前玩家**: "💬 你即将阵亡，请发表你的遗言。"
- **其他玩家**: "💬 等待即将阵亡的玩家发表遗言。"

玩家需要执行 `last_words` 动作，并提供遗言内容：

```json
{
  "action_id": "last_words",
  "params": {
    "content": "我是预言家，3号是狼人！"
  }
}
```

### 查看遗言历史

在游戏状态的 `current_state` 中：
- `last_words_history`: 包含当天所有已发表的遗言
- `last_words_pending`: 当前待发表遗言的玩家列表

## 技术特性

1. **无缝集成**: 遗言环节自动插入到所有死亡流程中
2. **多人支持**: 支持多个玩家同时发表遗言
3. **顺序保证**: 按照死亡顺序依次发表遗言
4. **状态一致性**: 遗言阶段不会影响游戏胜负判定
5. **猎人技能**: 正确处理猎人技能触发时的遗言流程

## 注意事项

1. 遗言内容不能为空
2. 只有即将死亡的玩家才能发表遗言
3. 遗言发表后立即生效，无法修改
4. 遗言阶段不计入游戏胜负判定
5. 如果猎人在遗言阶段后触发技能，会先处理猎人技能，再继续游戏流程

## 状态展示

在游戏状态栏中，遗言阶段显示为：
```
第 X 天 - 💬 遗言阶段
```

观战者视角：
```
👀 观战模式 - 第 X 天，遗言阶段。
```

## LLM玩家支持

### ✅ 完整支持

LLM玩家可以完整地感知和响应遗言功能：

#### 1. 信息接收

LLM玩家通过 `perspective.current_state` 接收遗言信息：

```json
{
  "phase": "day_discussion",
  "day": 2,
  "last_words_history": [
    {
      "day": 2,
      "speaker": "player_2",
      "content": "我是猎人，3号是狼人！大家相信我的判断。",
      "cause": "vote"
    }
  ]
}
```

#### 2. 记忆更新

LLM玩家可以自动将遗言信息记录到记忆中：

```json
{
  "action_id": "speak",
  "params": {
    "content": "player_2的遗言很重要..."
  },
  "memory_update": {
    "mode": "append",
    "content": "第2天：player_2（猎人）遗言指认3号是狼人，需要重点关注。"
  }
}
```

#### 3. 策略调整

LLM玩家会根据遗言内容调整行为：
- 好人阵营：根据神职遗言调整投票目标
- 狼人阵营：利用遗言混淆视听或调整战术
- 神职角色：根据遗言信息决定技能使用

#### 4. 发表遗言

当LLM玩家自己需要发表遗言时：

**LLM接收到的提示：**
```
# 当前游戏状态
{
  "phase": "last_words",
  "last_words_pending": ["player_3"]
}

# 可用行动列表
- action_id: "last_words"
  描述: 发表遗言
  参数:
    * content (string): 请输入你的遗言
```

**LLM可能的响应：**
```json
{
  "action_id": "last_words",
  "params": {
    "content": "我是预言家，昨晚查验5号是狼人，7号是好人。请大家相信我的判断，优先投5号！"
  },
  "reasoning": "作为预言家，我必须在死前传递关键信息"
}
```

### 技术细节

- **信息传递路径**: `lastWordsHistory` → `perspective.current_state.last_words_history` → LLM prompt
- **记忆系统**: 已启用（`enable_llm_memory: true`）
- **实时更新**: 遗言发表后立即对下一个行动玩家可见
- **历史记录**: 完整保留在 `lastWordsHistory` 中

### 应用场景

1. **预言家遗言**: LLM会记录查验结果并据此调整推理
2. **狼人遗言**: LLM狼人队友会根据遗言调整策略
3. **猎人遗言**: LLM会评估遗言可信度
4. **平民遗言**: LLM会综合分析遗言的合理性

详细说明请查看 `LLM遗言感知说明.md`
